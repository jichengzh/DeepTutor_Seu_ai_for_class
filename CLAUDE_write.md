# CLAUDE.md — 协作指南

## 任务执行流程

### 1. 开始前：制定计划
- 理解需求后，先简述实现思路和涉及文件，再动手
- 需求模糊时主动提问，不做大假设

### 2. 编码中：只做必要的事
- 不写兼容性代码，除非明确要求
- 不添加未被要求的功能、错误处理或抽象

### 3. 完成后：列出边缘情况与测试用例
每次完成一个功能后，主动列出：
- 正常路径的验证方式
- 边界值（空输入、最大值、零等）
- 可能的异常路径（网络失败、类型错误等）

然后**主动运行测试**，不等用户要求。

### 4. 出现 Bug：先写复现用例，再修复
```
发现 bug → 写最小复现测试 → 确认测试失败 → 修复 → 确认测试通过
```
不跳过复现步骤，即使 bug 看起来显而易见。

### 5. 每次纠错后：反思防范计划
修复错误后用一句话记录：
> **反思**：[错误原因] → [今后避免的具体做法]

例如：
> **反思**：忘记处理空列表导致 IndexError → 今后访问列表前先检查长度或用 `.get()`
